Day 6 – RNNs, LSTMs, and GRUs

1. Recurrent Neural Networks (RNN)

Concept: Standard neural networks assume that all inputs (and outputs) are independent of each other. However, for sequential data like text or time series, it's crucial to remember previous inputs. RNNs solve this by passing a hidden state from one time step to the next, acting as the network's "memory". The hidden state is updated based on both the current input and the previous hidden state.

2. Vanishing Gradient Problem & Temporal Dependencies

  • Vanishing Gradient: When backpropagating errors backward through time to update the network's weights, the gradients are continually multiplied by the weight matrices. If the weights are small, the gradients shrink exponentially and eventually "vanish", making it impossible to update weights for early layers.
  • Temporal Dependencies: Because the gradients vanish, basic RNNs cannot establish connections between events that are multiple time steps apart. If a sentence has a subject at the start and a verb 20 words later, a basic RNN will forget the subject by the time it reaches the verb.

3. Long Short-Term Memory (LSTM)

Concept: LSTMs heavily modify the standard RNN architecture specifically pattern to solve the vanishing gradient problem. They introduce a conveyor belt-like Cell State that runs straight down the entire chain, with only minor linear interactions. This allows information to flow along it unchanged. LSTMs then use three "gates" (layers with sigmoid activations that output numbers between 0 and 1) to optionally add or remove information.

  • Forget Gate: Looks at the previous hidden state and current input, and outputs a number between 0 and 1 to decide how much of the old memory (Cell State) to discard.
  • Input Gate: Decides what new information is going to be stored in the Cell State based on the current input.
  • Output Gate: Filters the newly updated Cell State, pushing it through a tanh activation, and outputs the next hidden state.

4. Gated Recurrent Unit (GRU)

Concept: The GRU is a streamlined, more efficient version of the LSTM. It gets rid of the separate Cell State entirely and uses only the hidden state to transfer information. Because it has fewer tensor operations, GRUs train faster than LSTMs while often achieving very similar performance.

  • Reset Gate: Acts a bit like the forget gate. It decides how much of the past information to forget before evaluating the new candidate state.
  • Update Gate: This gate combines the duties of the LSTM's forget and input gates. It decides both what to throw away and what new information to add at the same time.
