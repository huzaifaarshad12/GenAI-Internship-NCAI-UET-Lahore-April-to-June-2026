Day 7 – Transformer Architecture

1. Key Concepts Explained

  • Attention Mechanism: In sequence models, a bottleneck occurs if all context must be compressed into a single, fixed-size vector. The attention mechanism solves this by looking at all preceding tokens directly. It assigns a "weight" or "score" to every other word, indicating how much focus the model should place on it when understanding the current word.
  • Self-Attention: Specifically, this is how tokens interact within the SAME sequence. For the sentence "The bank of the river", when analyzing "bank", self-attention calculates high weights for "river" to deduce it means a geographical feature rather than a financial institution. It uses a mathematical operation involving learned Query (Q), Key (K), and Value (V) matrices.
  • Multi-Head Attention: Instead of performing just one attention operation, Transformers run many in parallel (multiple "heads"). Each head can learn to focus on different linguistic phenomena—for example, one head might track grammatical structure (subject-verb agreement), while another focuses on sentiment or pronoun antecedents.
  • Positional Encoding: Unlike RNNs that read words sequentially left-to-right, Transformers read entire sentences at once in parallel. This is incredibly fast but destroys all sense of word order. Positional encoding vectors (calculated using sine and cosine waves of different frequencies) are merged with the initial word embeddings. This gives the transformer a distinct mathematical signal about where each word is located absolutely and relatively.
  • Encoder-Decoder Structure:
    • The Encoder: Processes the entire input sentence. It passes data through multi-head self-attention and feed-forward layers to output a deep, highly contextualized representation of the original text.
    • The Decoder: Generates the output step-by-step. It uses "Masked" self-attention so it cannot cheat by looking at future words it hasn't generated yet. Importantly, it uses an "Encoder-Decoder Cross-Attention" layer, where its Queries check the Keys and Values from the final Encoder output to decide what translated word comes next.
